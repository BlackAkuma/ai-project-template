# QUICKSTART — ใช้งาน Template ใน 3 ขั้น

โฟลเดอร์นี้ใช้ **ครั้งเดียว** — AI อ่านแล้วสร้างระบบเอกสารให้ จากนั้นลบโฟลเดอร์นี้ทิ้งได้เลย

---

## ขั้นที่ 1 — เตรียมข้อมูล

| ข้อมูล | ตัวอย่าง |
|--------|---------|
| ชื่อโปรเจ็กต์ | `my-app` |
| path ที่จะสร้าง `doc/` | `/projects/my-app` |
| source docs | ไฟล์ PRD / spec / requirement ที่มี (ถ้ายังไม่มีพิมพ์ว่า "ยังไม่มี") |
| เป้าหมายสั้น ๆ | "แอปจองคิวออนไลน์สำหรับร้านอาหาร" |

---

## ขั้นที่ 2 — Copy Prompt นี้ให้ AI

> แทนค่าในวงเล็บ [ ] ก่อน copy

```
คุณกำลังจะ setup ระบบเอกสารสำหรับโปรเจ็กต์ใหม่

ข้อมูลโปรเจ็กต์:
- ชื่อ: [PROJECT_NAME]
- Path ที่จะสร้าง doc/: [PROJECT_ROOT_PATH]
- Source docs: [แนบไฟล์มาด้วย / ยังไม่มี]
- เป้าหมาย: [PROJECT_GOAL_SUMMARY]

ข้อกำหนด:
- ถามฉันก่อนว่าจะสื่อสารกันเป็นภาษาอะไร
- กระบวนการคิดภายใน: ใช้ภาษาอังกฤษเพื่อประหยัด token
- AI-CONTEXT block ในไฟล์ทุกไฟล์: ใช้ภาษาอังกฤษเสมอ
- output และเอกสาร: ใช้ภาษาที่ตกลงกัน

ขั้นตอน:
1. ถามภาษาที่จะใช้สื่อสาร รอคำตอบก่อน
2. อ่านไฟล์ทุกไฟล์ในโฟลเดอร์ template นี้ตามลำดับใน README.md
3. สร้างโครงสร้าง doc/ ที่ path ด้านบน
4. กรอกข้อมูลโปรเจ็กต์ที่มี ถ้าไม่พอให้ใส่ placeholder ชัดเจน ห้ามเดา
5. ตรวจสอบกับ 10-bootstrap-checklist-template.md ก่อนประกาศว่าเสร็จ
```

---

## ขั้นที่ 3 — รอ AI ทำงาน แล้วลบโฟลเดอร์นี้

AI จะ:
1. ถามภาษาที่ใช้สื่อสาร
2. อ่าน template ทั้งหมด
3. สร้าง `doc/` พร้อมไฟล์ทั้งหมดใน path ที่ระบุ
4. รายงานผลและตรวจ checklist

เมื่อ AI รายงานว่าผ่าน checklist แล้ว — **ลบโฟลเดอร์ template นี้ทิ้งได้เลย**

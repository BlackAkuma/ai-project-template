# Game Design Document (GDD) Template

เอกสารระดับ game — เขียนครั้งเดียวก่อนเริ่มโปรเจ็กต์ เก็บใน `CoreAiWorkspaces/00-source/versions/v0.1/gdd.md`
GDD คือ source of truth ของทิศทางทั้งเกม — FDD แต่ละตัวต้องอ้างอิงกลับมาที่ GDD เสมอ

AI ต้องอ่าน GDD ทุก session ก่อนทำงาน เช่นเดียวกับ source docs อื่น

---

## กฎการเขียน

AI เขียน GDD ทีละ section บันทึกไฟล์ แล้วรอคนยืนยันก่อนเขียน section ถัดไป
เมื่อเขียน section เสร็จ AI ต้องพูดว่า: **"Section [N] เสร็จแล้ว ยืนยันได้เลยหรือแก้ไขก่อน?"**

```md
## Section 4 — Core Loop

วงจรการเล่นที่ผู้เล่นทำซ้ำตลอดเกม — ต้องชัดเจนทั้ง 3 ระดับ

### Moment-to-Moment (วินาที → นาที)
ผู้เล่นทำอะไร → เกิดอะไร → ได้อะไรกลับมา

```
[Action] → [Feedback] → [Reward/Consequence] → [กลับไป Action]
```

### Session Loop (นาที → ชั่วโมง)
เป้าหมายในแต่ละครั้งที่เล่น — เริ่มยังไง จบยังไง รู้สึกยังไงตอนออก

### Progression Loop (วัน → สัปดาห์)
สิ่งที่เปลี่ยนแปลงในระยะยาว — unlock, สะสม, พัฒนาตัวละคร/ทักษะ

```
